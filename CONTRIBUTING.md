# Guía de Contribución

Cypher Vex exige los más altos estándares. Si deseas contribuir:

1. Haz un fork del repositorio.
2. Crea una rama para tu feature o bugfix (`git checkout -b feature/nueva-herramienta`).
3. Todo código debe estar estrictamente tipado y pasar la suite de pruebas.
4. Escribe tests en el directorio `tests/` para cualquier nueva funcionalidad.
5. Ejecuta `pytest` para verificar que tu código no rompe la estructura actual.
6. Realiza un PR detallando el objetivo, el problema que resuelve y adjunta evidencias.

*Cypher Vex no tolera código ineficiente.*
